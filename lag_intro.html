Link Aggregation (LAG): 聚合之力，打造高速且可靠的網路通道

探索如何將多條實體鏈路捆綁成一條邏輯通道，實現頻寬倍增與無縫備援

LAG：網路世界的多線道高速公路

🛣️ 什麼是 Link Aggregation？

Link Aggregation (LAG)，也常被稱為 EtherChannel 或 Port Trunking，是一種將多個獨立的實體網路連結（例如乙太網路埠）「捆綁」在一起，形成一個單一、高速的邏輯通道的技術。您可以想像成將數條鄉間小路拓寬、合併成一條多線道的高速公路。

🚀 為什麼需要 LAG？

  • 倍增頻寬：邏輯通道的總頻寬等於所有實體連結頻寬的總和。捆綁 3 條 1Gbps 的連結，您就能得到一條 3Gbps 的高速通道。
  • 提升可靠性 (備援)：如果捆綁群組中的某一條實體連結意外中斷，網路流量會自動且無縫地由其餘健康的連結接手處理，確保連線不中斷。
  • 負載平衡：LAG 會根據設定的演算法，智慧地將網路流量分散到群組中的所有實體連結上，避免單一連結過載，提升整體傳輸效率。

LAG 如何運作：LACP 的自動協商

在實務上，最常用來建立 LAG 的是 LACP (Link Aggregation Control Protocol)。LACP 允許Switch之間自動協商、建立並維護這個捆綁通道。兩端的設備會互相發送 LACPDU 封包，以確認彼此的狀態、監控連結的健康狀況，並動態地管理這個聚合群組。

在 LAG 之前，鏈路 1 是啟用狀態，鏈路 2 和鏈路 3 因 STP 而被阻斷。LAG 之後，所有鏈路捆綁成一條邏輯通道。
